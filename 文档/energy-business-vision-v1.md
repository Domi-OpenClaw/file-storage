**版本：v1.0 | 2026年4月**

---

## 一、公司战略定位

### 1.1 业务版图

| 业务板块 | 核心能力 | 定位 |
|---------|---------|------|
| **可信数据空间** | 数据确权、安全计算、交换标准、开放生态 | 底层基础设施 |
| **AI元能仓** | AI能力孵化、业务场景快速验证、数据产品快速迭代 | 创新加速器 |
| **能源数据产品** | 新能源消纳预测、电力现货价格预测、负荷曲线等 | 变现通道 |
| **售电网络** | 31省售电资质、数万工商业用户 | 客户渠道 |
| **储能运营** | 储能投资、运营管理、调度优化 | 运营变现 |

### 1.2 核心优势

**全国唯一：既有数据基础设施，又有客户渠道，还有AI能力，三位一体。**

- 竞争对手有售电，无数据空间基础设施
- 竞争对手有数据空间，无售电客户渠道
- 竞争对手有AI，无可信数据底座

## 二、可信数据空间

### 2.1 解决什么问题

| 行业痛点 | 我们的方案 |
|---------|-----------|
| 数据孤岛，无法打通 | 能源数据交换基础设施，互联互通 |
| 数据不敢共享，怕泄露 | 隐私计算+联邦学习，原始数据不出域 |
| 数据无法变现 | 数据产品化，上架交易 |
| 数据质量差 | 数据质量认证，标准化输出 |

### 2.2 核心架构

```
┌──────────────────────────────────────────────────────┐
│                  可信数据空间平台                      │
│                                                      │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐ │
│  │ 数据确权 │  │ 安全计算 │  │ 数据交换 │  │ 数据产品 │ │
│  │  存证链  │  │ 联邦学习 │  │  撮合引擎 │  │   商店   │ │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘ │
└──────────────────────────────────────────────────────┘
           ↓                    ↓
      发电侧数据             用电侧数据
     （新能源场站）         （工业/园区/充电）
```

### 2.3 数据产品三层体系

| 层级 | 产品 | 数据来源 | 定价方式 |
|------|------|---------|---------|
| **L1 原始数据** | 负荷曲线、发电曲线、用电画像 | 接入客户脱敏后 | 按端口/月 |
| **L2 智能数据** | 消纳预测、价格预测、负荷预测 | L1+AI模型 | 按次/按月 |
| **L3 决策数据** | 储能调度策略、购电优化方案、VPP聚合指令 | L2+业务模型 | 按项目/年 |

## 三、AI元能仓

### 3.1 定位

**可信数据空间内的AI能力孵化器**——快速将AI能力封装为可变现的数据产品，最小闭环验证后推广。

### 3.2 孵化流程

```
创意想法 → AI能力封装 → MCP工具 → 场景验证 → 数据产品 → 规模化推广
    ↓           ↓          ↓         ↓          ↓           ↓
  立项      算力+模型    API输出   客户试用   定价上架    复制扩张
```

### 3.3 孵化原则

- **最小闭环**：每个孵化项目3个月内要有付费验证
- **数据驱动**：用真实数据，不做PPT产品
- **可复用**：MCP工具孵化成功后在元能仓内共享，其他场景直接调用

## 四、核心数据产品

### 4.1 新能源消纳预测 MCP

**解决的问题**：明天各省市发多少电？哪些时段有富余？

**数据底座**：
- 天气：Open-Meteo API（全球免费天气数据）
- 装机：国家能源局2025年基准（31省光伏/风电装机容量）

**工具清单**：

| 工具 | 功能 | 覆盖 |
|------|------|------|
| `forecast_solar` | 光伏每日发电量预测 | 31省 |
| `forecast_wind` | 风电每日发电量预测 | 31省 |
| `forecast_combined` | 光伏+风电汇总 | 31省 |
| `forecast_curtailment` | 弃电率/弃电量估算 | 31省 |
| `forecast_with_storage` | 储能优化充放电策略 | 31省 |
| `check_alerts` | 全网消纳风险扫描 | 31省 |
| `compare_provinces_curtailment` | 多省消纳风险排名 | 31省 |
| `generate_daily_digest` | 每日简报（人读版） | 全国/分省 |
| `list_provinces` | 31省装机容量查询 | 31省 |

## 五、商业模式

### 5.1 三大收入来源

| 模式 | 说明 | 收入特征 |
|------|------|---------|
| **数据产品销售** | 消纳预测、现货价格预测等产品按次/按月收费 | 订阅制，续费率高 |
| **平台佣金** | 数据空间撮合交易，抽取10-20% | 轻资产，规模化快 |
| **运营分成** | 储能运营、售电协同，与客户按比例分成 | 重资产，高客单价 |

### 5.2 储能运营盈利测算

## 六、落地路径

### 6.1 三阶段规划

**第一阶段：孵化验证（1-6个月）**

| 动作 | 目标 |
|------|------|
| MCP产品上架数据空间 | 2个MCP工具封装为可调用API |
| 首批客户试用 | 浙江/广东签约3-5家工业客户 |
| 储能项目落地 | 10 MWh储能试点项目并网 |
| 数据产品定价 | 确定订阅价格，验证付费意愿 |

**第二阶段：规模扩张（6-12个月）**

| 动作 | 目标 |
|------|------|
| 储能规模复制 | 覆盖浙江/广东，新增上海/广西，装机100 MWh |
| 售电协同深化 | 签约200家工商业客户 |
| 数据产品矩阵 | 新增负荷预测、VPP调度指令等产品 |
| 平台撮合上线 | 数据交易撮合平台正式运营 |

**第三阶段：生态构建（12个月+）**

| 动作 | 目标 |
|------|------|
| 第三方开发者引入 | 开放API，吸引合作伙伴基于MCP开发应用 |
| 能源数据交易所对接 | 数据产品进入官方数据交易所 |
| 全国复制 | 从长三角/珠三角向其他省份复制 |
| AI能力扩展 | 新增负荷预测AI、火电调度AI等MCP |

### 6.2 试点省份选择

| 优先级 | 省份 | 理由 |
|--------|------|------|
| 🔴 第一 | **浙江** | 峰谷价差1.32元/kWh最优，深谷政策加持，售电市场成熟 |
| 🔴 第二 | **广东** | 峰谷价差1.32元/kWh，珠三角工业密集，尖峰溢价高 |
| 🟡 第三 | **江苏** | 需量优化空间大，客户基础好，峰谷价差收窄后需多元盈利 |

|------|------------|------------|------|
